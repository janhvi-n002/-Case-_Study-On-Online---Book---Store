SQL Case Study: 
 ## 1. Books Table
This table contains information about books available in the store.

 # Columns:
- Book_ID: Unique identifier for each book
- Book_Name: Name of the book
- Author: Author of the book
- Genre: Category of the book
- Price: Price of the book
## 2. Customers Table
This table stores customer details.

 # Columns:
- Customer_ID: Unique identifier for each customer
- Customer_Name: Name of the customer
- Email: Email address
- City: City of the customer
 ## 3. Orders Table
This table records all purchase transactions.

# Columns:
- Order_ID: Unique identifier for each order
- Customer_ID: Customer who placed the order
- Book_ID: Book purchased
- Order_Date: Date of order
- Quantity: Number of books ordered
 ## Conclusion
This case study helps in practicing SQL concepts such as JOINs, GROUP BY, aggregate functions,
subqueries, and date functions using real-world styled data.
